# 飘流幻境新世界 - 智能炼金模拟与截图收集服务设计文档

本项目由两部分核心功能构成：
1. **智能炼金模拟器 (Alchemy Simulator)**：支持多槽材料放置、百科设定、实时期望概率预测及批量连续合成概率仿真的精美前端应用。
2. **合成截图收集服务 (Uploader Service)**：提供极简的拖拽 / 粘贴直传腾讯云 COS 存储截图，并在本地记录上传日志的 Go 后端基础服务。

## 🔮 1. 炼金核心预测算法 (`alchemy_core.js`)

共享计算逻辑包含完整的官方高阶炼金模拟规则，支持 2~5 个插槽的多物品融合。

### A. 等级计算公式
1. **基准等级 ($L_{\text{base}}$)**：
   $$L_{\text{base}} = \min(L_1, L_2, \dots, L_N)$$
   其中 $L_1$ 为主插槽（槽1）物品等级，其余为副材料等级。
2. **物等波动区间**：
   - 默认下限 $L_{\text{min}}$：若 $L_{\text{base}} \ge 8$，则下限为 $L_{\text{base}} - 7$，否则为 $1$。
   - 默认上限 $L_{\text{max}}$：$L_{\text{base}} + 4 + B$，其中 $B$ 为百科书加成级数（无百科为 $0$，百科一至四对应 $+1 \sim +4$）。

### B. 级别跳跃（Level Jump）概率分布
系统在区间内进行随机物等跳跃，概率权重如下：
* **+4 级** (最高物等上限)：10%
* **+3 级**：25%
* **+2 级**：30%
* **+1 级**：15%
* **+0 级** (基准等级)：15%
* **降级 (小于基准等)**：共 5%（平均分配到基准线以下最多 7 个级别）。

### C. 产物候选查找与 Fallback 降级匹配
1. **过滤规则**：
   - 材质必须与主材料（槽1）相匹配或相容。
   - 分类属于可合成品（如装备或矿物，req_level > 0）。
   - 主属性不能发生反偏离（如槽2~5物等大于槽1主材可能引发吃属警告）。
2. **回退兜底**：
   若计算出的目标物等在数据库中没有任何合法候选：
   - **优先向下寻找**：以目标物等为起点，逐级向下寻找最近有候选物品的级别。
   - **向上寻找**：若下方直至1级都无候选，则向上寻找最近有候选的级别。
   - 找到的第一个有候选的级别即作为该目标的最终 fallback 代表项。

## 📤 2. 截图收集与大模型 OCR 标注服务 (`uploader_service`)

采用 **服务端代理直传** 模式，保证了腾讯云密钥的安全，同时避免了前端跨域及存储桶 CORS 配置的繁琐步骤。用户上传图片后，通过本地 Python 脚本提取并使用大模型识别，最后回填结构化数据。

### A. 运行配置 (`config.json`)
```json
{
  "tencent_cos_secret_id": "您的SecretID",
  "tencent_cos_secret_key": "您的SecretKey",
  "tencent_cos_bucket": "您的存储桶名称-APPID",
  "tencent_cos_region": "ap-guangzhou",
  "port": "8080"
}
```

### B. 数据库结构升级 (`uploads.json`)
数据库支持记录识别状态与大模型回填的多条结构化配方结果（每张截图通常垂直包含 3 条结果）：
* `status`：`"pending"` (待识别) 或 `"recognized"` (已识别)。
* `result`：包含 `recipes` 数组的识别数据，未识别时为 `null`。

```json
[
  {
    "id": 1,
    "timestamp": "2026-07-04T07:35:39+08:00",
    "image_url": "https://pln-pic-1259639420.cos.ap-guangzhou.myqcloud.com/images/1783121739404_clipboard_1783121738538.png",
    "status": "recognized",
    "result": {
      "recipes": [
        {
          "target_name": "白玉",
          "target_level": 5,
          "slot1_name": "白玉",
          "slot1_level": 5,
          "slot2_name": "粉水晶",
          "slot2_level": 5,
          "book": 0,
          "slot1_material": "玉",
          "slot2_material": "水晶"
        }
      ]
    }
  }
]
```

### C. 大模型标注数据脚本流
* **拉取输出脚本 (`export_pending.py`)**：
  读取 `uploads.json`，筛选所有 `"status": "pending"` 的记录，在控制台输出图片 URL 列表，并自动在本地生成包含未识别图片及提示词模板的 `prompt.txt`。您可以直接复制该提示词与图片外链发给 Vision 大模型（如 GPT-4o, Claude 等）进行识别。
* **回填录入脚本 (`import_results.py`)**：
  读取大模型生成的 `results.json`。遍历每张图的识别配方，自动关联上一级目录的 `alchemy_db.json` 以**自动补全主副材料的材质属性**（如将主材料 `白玉` 补全为 `"slot1_material": "玉"`，副材料 `粉水晶` 补全为 `"slot2_material": "水晶"`），随后回填写入数据库，修改状态为 `recognized`，并将结果文件重命名归档为 `results_imported.json`。

### D. 服务端 API 设计
* `POST /api/upload`：
  - 前端以 `MultipartForm` 格式提交键为 `file` 的文件流。
  - 后端在内存解析，生成基于时间戳的安全文件名：`images/时间戳_文件名.ext`。
  - 调用腾讯云 COS Go SDK 将文件直传至目标存储桶，并赋予 `public-read` 权限。
  - 完成上传后，将生成的外链记录（默认状态为 `"pending"`，结果为 `null`）原子性写入本地 `uploads.json` 并返回最终访问地址。
* `GET /api/upload/records`：
  - 读取并返回所有已收录的合成截图列表（前端会根据记录的识别状态，在卡片上直接显示提取出来的各条合成路径）。

### E. 粘贴直传 (`Ctrl + V`)
- 前端 [upload.js](file:///d:/飘流幻境新世界/alchemy_simulator/upload.js) 监听了全局的 `paste` 事件。当玩家在游戏内按 `PrintScreen` 截图后，在网页直接 `Ctrl + V` 即可瞬间载入图片并进行预览直传，省去保存到磁盘再选文件的繁杂步骤。

## 🛠️ 3. 运行指南

### A. 启动 Go 服务端
1. 打开命令行，进入服务端目录：
   ```bash
   cd uploader_service
   ```
2. 运行 Go 服务：
   ```bash
   go run main.go
   ```
3. 控制台会输出启动监听地址：
   ```
   Uploader Server starting on http://localhost:8080 ...
   Access upload portal directly at: http://localhost:8080/alchemy_simulator/upload.html
   ```

### B. 打开应用
- **合成模拟器**：访问 [http://localhost:8080/alchemy_simulator/index.html](http://localhost:8080/alchemy_simulator/index.html)
- **截图收集库**：访问 [http://localhost:8080/alchemy_simulator/upload.html](http://localhost:8080/alchemy_simulator/upload.html)
